# Monitoring Implementation Details

## Amazon SNS Topic Configuration

![SNS Topic](centralized-logging/screenshots/1-sns-topic.png)

An Amazon SNS topic named **EC2-Alerts** was created to handle notification delivery for CloudWatch alarms. This topic acts as a central communication channel between monitoring services and subscribed endpoints.

---

## Email Subscription Setup

![SNS Subscription](centralized-logging/screenshots/2-sns-subscription.png)

An email endpoint was subscribed to the SNS topic. Whenever alarms are triggered, notifications are delivered automatically to the configured email address.

---

## Subscription Confirmation

![Subscription Confirmation](centralized-logging/screenshots/3-subscription-confirmed.png)

After confirming the email verification request, the SNS subscription status changed to **Confirmed**, enabling alert delivery.

---

## EC2 Monitoring Configuration

![EC2 Instance](centralized-logging/screenshots/4-ec2-instance.png)

An EC2 instance was selected for infrastructure monitoring. CPU metrics generated by the instance were used for alarm creation and dashboard visualization.

---

## CPU Alarm Configuration

![CPU Alarm](centralized-logging/screenshots/5-cpu-alarm.png)

A CloudWatch alarm named **High-CPU-Alarm** was created to monitor EC2 CPU utilization.

Alarm configuration:

- Metric: CPUUtilization
- Threshold: Greater than 70%
- Evaluation period: 5 minutes

---

## CPU Alarm Details

![CPU Alarm Details](centralized-logging/screenshots/6-cpu-alarm-details.png)
